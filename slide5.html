Configuration Management

Key Design Principle:

Enable configurability to facilitate easy experimentation.

Features:

  • Centralized settings for all pipeline components
  • Enables easy experimentation without code changes
Data Ingestion, Cleaning and Preprocessing

Key Design Principle:

Offers the capability to choose various features and determine their preprocessing approaches.

Functions:

  • Comprehensive and configurable data cleaning and feature engineering
  • Choose core features for model and how to preprocess them
  • Select engineered features for experimentation based on insights gained from EDA
Training & Evaluation

Key Design Principles:

1. Create unbiased performance estimates with nested cross validation
2. Automated selection of best model and perform threshold optimization
3. Allow ease of evaluation with proper recording and visualization techniques

Capabilities:

  • Configurable splits for nested cross validation
  • Usage of GridSearchCV for hyper parameter tuning
  • Save evaluation results to csv for easy access and evaluation
Scikit-learn Pipeline Creation

Key Design Principle:

Automate and streamline pipeline creation

Features:

  • Dynamic feature type detection (categorical vs numeric) based on selected features
  • Automated preprocessing pipeline construction based on selected models
  • Seven model implementations to choose from for experimentation

  • Automated visualization generation
